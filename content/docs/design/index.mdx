## Design Philosophy

### 1. User-Focused Architecture

Technical decisions should improve user experience. Complex state machines exist to provide clear, consistent feedbackâ€”not to satisfy engineering elegance.

### 2. Explicit Over Implicit

State transitions, error handling, and user messaging should be explicit and documented. Hidden behaviors create debugging nightmares.

### 3. Graceful Degradation

When systems fail (and they will), users should understand what happened and have clear paths forward. Offline doesn't mean broken.

### 4. Platform Conventions

Follow iOS and web platform conventions. Users shouldn't need to learn new interaction patterns for familiar concepts.
